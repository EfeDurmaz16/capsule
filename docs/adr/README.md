# Architecture Decision Records

Capsule uses ADRs for decisions that shape the public SDK contract, adapter model, provider boundary, or maintenance strategy.

These records are intentionally short and explicit. They capture what Capsule chooses, what it rejects for now, and what the project gains or loses from that choice.
